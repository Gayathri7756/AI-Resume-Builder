## Documentation Principles

These docs follow these principles:

1. **Progressive Disclosure** - Start simple, go deep
2. **Task-Oriented** - Organized by what users want to do
3. **Visual** - Diagrams and examples over walls of text
4. **Practical** - Real commands, real examples
5. **Complete** - Cover happy path AND edge cases
